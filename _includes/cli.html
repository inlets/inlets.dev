uplink.example.com

Connect a customer environment to your SaaS

Create a tenant-scoped tunnel, hand the customer a command, then consume the service privately from inside your cluster.

# tunnel resource
apiVersion: uplink.inlets.dev/v1alpha1
kind: Tunnel
metadata:
  name: acmeco
  namespace: tunnels
spec:
  tcpPorts:
    - 8080

# customer connects to the router
inlets-pro uplink client \
  --url wss://uplink.example.com/tunnels/acmeco \
  --upstream http://127.0.0.1:8080 \
  --token-file ./token.txt

# private access from inside the cluster
kubectl run -t -i curl --rm \
  --image ghcr.io/openfaas/curl:latest /bin/sh

curl -i http://acmeco.tunnels:8000

api.example.com

Self-host HTTPS for a local app

Provision a public tunnel server with inletsctl, then connect your local HTTP service using your own domain.

$ inletsctl create \
  --provider digitalocean \
  --region lon1 \
  --access-token-file ~/digital-ocean-api-key.txt \
  --letsencrypt-domain api.example.com

$ inlets-pro http client \
  --url wss://178.62.70.130:8123 \
  --token-file ./token.txt \
  --upstream "api.example.com=http://127.0.0.1:3000"

$ curl "https://api.example.com/v1/import"

178.62.70.130

Expose multiple TCP ports without SaaS tunnel limits

Provision a TCP tunnel VM with inletsctl, then connect local services such as webhooks or SSH over the same tunnel.

$ inletsctl create \
  --provider digitalocean \
  --region lon1 \
  --access-token-file ~/digital-ocean-api-key.txt \
  --tcp

$ inlets-pro tcp client --url wss://178.62.70.130:8123 \
  --token-file ./token.txt \
  --upstream "192.168.0.35" \
  --ports "8080,2222"

$ curl "http://178.62.70.130:8080/function/webhook"
$ ssh -p 2222 "pi@178.62.70.130"

kubernetes / loadbalancer

Expose a local Kubernetes service

Install the inlets-operator Helm chart, then create a LoadBalancer service and wait for a public IP.

# Install the inlets-operator Helm chart, then:
$ kubectl run nginx-1 --image=nginx --port=80 --restart=Always
$ kubectl expose pod/nginx-1 --port=80 --type=LoadBalancer

$ kubectl get services -w
NAME              TYPE           CLUSTER-IP         EXTERNAL-IP   PORT(S)   AGE
service/nginx-1   LoadBalancer   192.168.226.216         80/TCP    78s
service/nginx-1   LoadBalancer   192.168.226.216    104.248.163.242   80/TCP    78s

$ curl -s "http://104.248.163.242:80/"
